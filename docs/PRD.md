## 3. 필수 기능 명세 (User Stories)

### 3.1 포트폴리오 웹 페이지 (방문자 대상)

| ID       | User Story                                                                     | Priority   | Acceptance Criteria                                                      |
| :------- | :----------------------------------------------------------------------------- | :--------- | :----------------------------------------------------------------------- |
| USR-001  | 방문자로서, 사이트 소유자의 이름과 직함을 한눈에 볼 수 있는 히어로 섹션을 보고 싶다 | **Must**   | HeroSection 렌더링, 이름/직함/CTA 표시, 이력서 다운로드 동작             |
| USR-002  | 방문자로서, 소유자의 배경과 경험을 이해할 수 있는 자기소개 섹션을 보고 싶다     | **Must**   | AboutSection 렌더링, 프로필 이미지 + 소개 텍스트                         |
| USR-003  | 방문자로서, 소유자의 기술 스택을 카테고리별로 확인하고 싶다                     | **Must**   | SkillsSection 렌더링, 카테고리별 그리드                                  |
| USR-004  | 방문자로서, 소유자의 프로젝트 포트폴리오를 필터링하여 보고 싶다                 | **Must**   | ProjectsSection 렌더링, 카테고리 필터, 카드 클릭 시 상세 이동            |
| USR-005  | 방문자로서, 소유자의 경력/학력을 타임라인으로 보고 싶다                         | **Should** | ExperienceSection 렌더링, 타임라인 UI                                    |
| USR-006  | 방문자로서, 소유자에게 연락 메시지를 보내고 싶다                               | **Must**   | ContactSection 렌더링, 폼 검증, 이메일 발송, 성공/실패 피드백            |
| USR-007  | 방문자로서, 다크/라이트 모드를 전환하고 싶다                                   | **Should** | 토글 버튼, 시스템 설정 감지, 선택 기억 (localStorage)                    |
| USR-008  | 방문자로서, 모바일에서도 편하게 사이트를 탐색하고 싶다                         | **Must**   | 반응형 레이아웃, 모바일 네비게이션 드로어                                |
| USR-009  | 방문자로서, 프로젝트 상세 페이지에서 스크린샷과 기술 설명을 보고 싶다          | **Could**  | `/projects/[slug]` 라우트, 상세 정보 렌더링                              |
